手把手教学搭建hexo博客

全流程手把手教学搭建hexo博客

本地搭建hexo

  1. 下载nodejs，进行本地安装，使用node -v和npm -v确认

  2. 安装hexo，使用npm install -g hexo-cli，使用hexo -v确认

  3. 创建博客目录，比如g:/project/blog，blog目录就作为hexo的工作目录，其博客内容与hexo的生成与部署都将在此完成

  4. 进入博客目录并初始化博客目录，cd g:/project/blog，hexo init

  5. 启动博客，hexo start或hexo server，默认在本地4000端口启动，此时即可在本地访问，比如http://localhost:4000

  6. 新建一篇博客，hexo new "我的第一篇博客"，即可在./source/_posts目录下看到新生成的md文件，从而可以撰写博客

  7. 清理并生成html文件，hexo clean，hexo generate

  8. 再次启动博客，hexo server

  9. 到此本地博客搭建完成

    部署到github

  10. 本地博客只能在本地访问，不够方便，hexo支持部署到github进行访问

  11. 在github中新建一个repo，其名称格式是[用户名].github.io，比如qirangit.github.io

  12. 本地电脑安装hexo的部署git的插件，npm install --save hexo-deployer-git

  13. 进入blog目录，修改配置文件，notepad _config.yml，修改底下的deploy参数，将部署type改成git，将repo名称改为github的repo地址，将branch推送分支改为master，比如

  14. 将生成的html部署到github，hexo deploy

  15. 这样通过访问repo地址的即可访问博客，比如https://github.com/qirangit/qirangit.github.io

    部署到云服务器

  16. 由于部署到github上无法被检索，另外访问者体验较差，所以可以部署到云服务，并通过域名快捷访问

  17. 原理，简单来说就是首先在本地搭建Hexo，通过hexo generate将md文件渲染成静态的html页面，然后通过hexo deploy触发设置用户通过公钥免密登陆服务器，进而将静态页面推送到云服务器的git仓库中，然后服务器再通过钩子（git-hooks） 将静态页面同步到网站的根目录下，进而实现博客的自动部署。

  18. 从阿里云购买云服务器，获得服务实例

  19. 阿里云默认限制80端口访问，解除阿里云的访问限制，手动添加安全组为端口和ip放行，具体操作是创建”创建安全组”进行解除，里面需要将端口范围改成1/65535，授权对象改为0.0.0.0/0。

  20. 解除linux服务器的80端口限制(ubuntu不限制80端口)，/sbin/iptables -I INPUT -p tcp –-dport 80 -j ACCEPT

  21. 安装nginx，这里选择使用nginx作为http服务器，来访问博客内容。

  22. 使用docker安装nginx，docker pull nginx

  23. 检查云服务器，启动nginx，docker run -d --name mynginx -p 80:80 -v /home/www/hexo:/home/www/hexo nginx，这里数据卷目录用于存放博客项目内容，此时通过访问服务器的ip加80端口，能够看到默认的nginx欢迎界面

  24. 修改nginx配置，使其访问博客内容，进入容器，docker exec -it mynginx /bin/bash，修改配置文件，vim /etc/nginx/conf.d/default.conf，修改域名server_name，默认是localhost，改为自己的域名，比如www.robotqi.cn，如果没有域名，先写公网ip；修改第一个location下的root项目访问目录，/home/www/hexo。(nginx容器中没有安装vim，需要手动安装)

  25. 创建hexo博客的部署目录，mkdir /home/www/hexo(若前面启动docker容器已经创建不用新增)

  26. 服务器安装nodejs，apt-get update，apt-get install nodejs，apt-get install npm，安装git，apt-get install git

  27. 为了实现博客自动部署，需要设置公钥免密登录服务器。做法是本地生成公钥，然后通过ftp传递到服务器中，然后将其内容复制到authorized_keys文件中，这样就能实现ssh免密登录服务器

  28. 在服务器的用户根目录下创建.ssh文件夹，cd /root，mkdir .ssh

  29. 通过ftp工具将本地的id_rsa.pub文件传递到服务器的.ssh目录下，并且将其内容copy到authorized_keys文件中，cp id_rsa.pub authorized_keys

  30. 确保设置了正确的SELinux上下文，restorecon -Rv ~/.ssh

  31. 此时在本地使用ssh连接服务器不会提示输入密码，ssh -v root@xxx.xxx.xxx.xxx(阿里云公网IP)

  32. 在服务器中创建git仓库，新建钩子文件，cd /root，git init --bare hexo.git，vim root/hexo.git/hooks/post-receive，文件内容是git --work-tree=/home/www/hexo --git-dir=/root/hexo.git checkout -f，表示让钩子文件删除/home/www/hexo目录下原有的文件，然后从blog.git仓库 clone 新的博客静态文件到/home/www/hexo目录下。

  33. 重启云服务器，使上述配置生效。

  34. 修改本地blog目录下_config.yml中的deploy参数，将其repo地址改成云服务器端的repo地址

    然后使用自动部署，hexo-deployer-git --save

  35. 使用hexo进行生成和部署，hexo clean，hexo generate和hexo deploy，此时就能通过浏览器访问个人博客了
