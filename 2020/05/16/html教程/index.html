html教程

web发展历史

  1. web前端的技术发展经历过很多时期，注意以下时期并不是泾渭分明，其实是相互交错，并且技术也是继承与发展，而不是简单的更新换代
    1. 静态时期，静态是指html页面已经提前创建生成，此时通常通过nginx或apache等http服务器绑定80端口来管理这些静态页面，供前端请求。静态页面由于部署简单方便，现在还比较常见
    2. 动态时期，此时前端请求到后端时，服务端会根据http的请求内容，动态地生成html页面，这时衍生出很多种服务端的html页面渲染技术，
      1. CGI，这个在服务端直接拼接出html页面字符串，由于效率低下，现在已经淘汰
      2. 模板技术，常见的有jsp、asp和freemarker等，比如在java服务端场景下，常使用tomcat+jsp搭建，这时服务端通过搭配模板可以较快地生成html页面，由于实现方便，所以在简单场景下模板技术还时常使用
    3. js脚本时期，原有的前端页面如果想要实现加载更新，就只能访问后端，这时衍生出了web使用的脚本比如javascript，浏览器通过执行js脚本，配合dom可以在前端就实现页面更新效果，搭配ajax可以实现不阻塞当前页面的加载更新
    4. 前后端分离时期，原有的html页面都是在服务端渲染，前端获取的都是html页面数据，此后随着前端项目的日益复杂，前端走向独立的工程化道路，开始前后端分离，此时服务端不提供view层功能，前端不再直接获取html页面而是开始获取服务端的json数据，前端通过配合vue或react等js框架在前端完成html页面的渲染，至此前后端直接的交互变成了以json数据为主，类似于后端微服务的交互，也需要定义接口契约，前端开始走上独立发展的道路

html学习准备

  1. 可以在idea或者webstorm等ide里新建一个项目，在根目录下创建index.html，使用chrome浏览器通过file协议通过某个端口打开html文件，即可看到渲染后的html页面

  2. html是一个超文本标记语言，其是一种DSL来描述网页内容结构，标签和文本共同组成了html网页，html元素表示从开始标签到结束标签的所有代码，html元素分为两种

    • 一种有元素内容，此时包括三部分，分别是开始标签、元素内容和结束标签(正斜杠位于标签字母前)

    • 另外一种是空元素，其并没有元素内容和结束标签，其只有开始标签，并且在开始标签的标签字母后有正斜杠/，比如换行<br/>

html基础知识

  1. html标签通过属性来提供更多标签功能，属性是通过键值对形式出现，键直接用属性名表示，值必须是字符串，键与值之间用=分隔，键值对之间用空格分隔，属性分为特有属性和通用属性，

    1. 通用属性是每个标签都拥有的，比如类名class(用于分类)、id(用于标识)、样式style
    2. 特有属性属于特定标签自身的，比如bgcolor属性，可用于设置背景色，align属性表示对齐方式(样式类的属性现在不建议标签内使用)，a标签的target表示超链接跳转的方式

  2. 首先了解一个html文档的总体结构

    1. doctype说明，html代码首先需要声明html的版本，现在都是使用2012年诞生的html5，所以都是声明为<!DOCTYPE html>
    2. 由于html网页是一个树形结构，所以总体呈现的是嵌套结构，最外围的标签就是html，html标签自身需要声明一个属性lang即lang=en，然后里面包含着head和body标签
    3. head标签里面包含一些声明，比如字符集<meta charset="UTF-8">，网页名称<title>我的网页</title>
    4. body标签里的内容会被浏览器渲染展示出来，所以具体的展示的标签内容在此嵌套书写

  3. 常见标签

    1. h1~h6标签用于标题展示，标题与段落块级标签不仅自身独占一行，前后还空一行

    2. p标签用于段落展示

    3. br标签用于换行，hr标签用于水平线

    4. 格式化标签，

      1. 粗体b标签
    2. 斜体i标签
    3. 下划线ins标签
    4. 删除线del
    5. 代码标签code
    6. 引用标签q和cite

    5. a标签用于超链接，比如<a href="http://www.baidu.com">百度网站</a>，这里在点击之后使得浏览器发送get请求请求该链接，比如<a href="jump.html">jump页面验证</a>，这里点击后就会跳转到根目录下的jump.html页面

    6. img标签用于图片展示，我们在根目录下创建images目录，存放图片文件，

      1. 使用src属性引入，比如<img src="images/w3c.jpg" >

      2. 还有alt属性，图片描述文字，图片无法显示时会提示

      3. 包括图片文件路径在内的路径分为两种，分别是绝对路径和相对路径

        • 相对路径，就是相对当前文件的路径，推荐使用

        • 绝对路径，根据网站根目录而来，根目录首先得确定出来才行，比如

    7. html的样式有三种设置方式，

      1. 外部样式表，也就是css文件单独抽取出来，这里需要使用link标签，比如<link rel="stylesheet" type="text/css" href="css/outer.css">

      2. 内部样式表，这里使用style标签，其元素内容就是css代码，比如

      3. 内联样式，这里使用普通标签的通用属性style，直接定义样式，比如<p style="color: yellow;size: portrait">内部属性</p>，这里样式也是使用键值对，键与值使用:分隔，键值对之间使用;分隔

    8. a标签

      1. 其元素内容可以是文本，也可以是图片，或者其他html元素都可以，比如<a href="index.html"><img src="images/w3c.jpg"></a>
      2. href即指向文档的连接，可以指定一个网络上的url地址，也可以指定本地的网页
      3. target属性表示文档被打开在何处，常见的值有_blank

    9. name属性(id属性也可以)用于设置当前a标签为一个锚点，其他的a标签通过href设置就可以跳转到该锚点，比如有一个锚点<a name="hello">z这是一个hello锚点</a>，跳转的链接<a href="#hello">点击跳转到hello锚点</a>

    10. table标签

      • table标签，其下有caption表格标题标签和tr行标签，其还拥有很多属性，比如border边框宽度、cellpadding单元格内边距、cellspacing单元格外边距

      • tr标签下有th标签和td标签，th标签表示列头单元格，td标签表示普通单元格

11. 列表标签，分为有序列表ol和无序列表ul

     1. 有序列表ol，列表选项li，默认使用数字，可以通过ol标签的type属性设置为i或者a，另外可以设置start属性，表示开始的数字

     2. 无序列ul表，列表选项也是li，默认形状是空心圆disc，可以通过type属性设置为circle实心圆或者square实方形

     3. 列表可以进行嵌套，从而实现多层级，这里嵌套的方式就是直接在需要展开的li标签后面写上新的ul或ol列表即可

        

12. 块元素
     1. html中标题h1、段落p、列表ul和div都是块级元素(block element)，块级元素的前后保证换行，其中div作为一种块级容器存在
     2. 除了块级元素外，其他的就是内联元素(inline element)，其并不会换行，比如a、b和span标签，其中span作为一种内联容器存在

13. 布局
     1. 网站页面需要布局，通常来说一个网站有五块区域组成，分别是顶栏、左侧栏、内容、右侧栏和底栏，可以使用div布局或者table布局
     2. 使用div布局，需要设置包括总体的六个div，然后每个div都需要对其宽高进行设置，并且还需设置float和clear属性
     3. 使用table布局，说的是用表格来布局，其思路是通过tr和td进行布局，由于表格用于格式化，一般不适合布局
     4. 背景颜色设置使用bgcolor属性，背景图片设置使用background属性
             

14. 框架，这里的框架是指嵌套另一个页面，这里使用iframe标签，其src属性表示嵌套的网页，这里需要注意的a标签的target属性，有若干个值，在这种嵌套的场景下，其中_self表示仅在自己所属的frame中打开，_blank表示新的tab打开，_parent表示在父frame打开，而_top则是从当前的tab重新打开

16. 实体，html中一些符号是作为关键字存在，如果想要显示它们，理所当然需要转义，这里对于`<`使用`&lt;`，对于`>`使用`&gt;`，对于连续的空格，由于html会自动截短至一个空格，所以使用`&nbsp;`

17. 表单，表单不同于以上的标签，其可以用于输入，表单标签是form，其内部还嵌套多种类型的输入标签，

      1. input标签，其用于输入，其为空元素标签，通过type可以指定输入类型，比如text文本、password密码、checkbox多选框、单选框radio、submit提交，reset重置和button普通按钮，这里按钮的名称通过value进行指定(按钮可以通过单独的button标签来表示，其type是submit或reset，其按钮名称则是通过闭合的元素内容表示，了解即可，不推荐使用)

      4. form拥有action属性表示提交的url，当提交时需要指定请求的页面，默认是当前url的页面，另外method表示提交的方法，默认是get方法，如下就会将表单通过参数形式提交到demo_form.asp页面中去

xhtml介绍

  1. xhtml是通过xml来编写html，由于html松散的语法规范，使得存在很多badcode，而由于xml的严格的规范特性，使得使用xhtml编写网页代码会更加严谨规范，目前xhtml得到浏览器的广泛支持，xhtml中的特性列举如下，
    • 文档声明，XHTML <!DOCTYPE>，此时该网页就是xhtml网页
    • 标签元素必须小写且必须嵌套，并始终关闭
    • 标签的属性必须小写，属性值必须使用引号包围，另外属性不得简写，比如<input checked />是错误的，<input checked="checked" />是正确的

html5介绍

  1. html5是最新的html标准，其标签属性更加丰富

    • 添加了一些表单类型比如时间、视频标签video和语义标签等，其中语义标签的意思是原有的诸如于div和span标签都是非语义的，不表示任何含义，通常在网页布局时，会通过id属性来标识div各个作用区域，在html5通过语义标签，能从标签名称就能知道标签内容的作用与含义，比如article、header、nav、footer、aside、section等标签
    • 添加一些新通用属性，比如contentEditable，表示是否可编辑，hidden表示隐藏，其页面不可见，spellcheck属性表示会对输入内容进行检查

  2. html最佳实践，任何最佳实践都是为了规范统一，方便阅读，对于代码的要求不仅仅是可运行，更在于可阅读和可维护，最佳实践请见参考文档
