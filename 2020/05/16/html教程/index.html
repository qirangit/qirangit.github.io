html教程

web发展历史

  1. web前端的技术发展经历过很多时期，注意以下时期并不是泾渭分明，其实是相互交错，并且技术也是继承与发展，而不是简单的更新换代
    1. 静态时期，静态是指html页面已经提前创建生成，此时通常通过nginx或apache等http服务器绑定80端口来管理这些静态页面，供前端请求。静态页面由于部署简单方便，现在还比较常见
    2. 动态时期，此时前端请求到后端时，服务端会根据http的请求内容，动态地生成html页面，这时衍生出很多种服务端的html页面渲染技术，
      1. CGI，这个在服务端直接拼接出html页面字符串，由于效率低下，现在已经淘汰
      2. 模板技术，常见的有jsp、asp和freemarker等，比如在java服务端场景下，常使用tomcat+jsp搭建，这时服务端通过搭配模板可以较快地生成html页面，由于实现方便，所以在简单场景下模板技术还时常使用
    3. js脚本时期，原有的前端页面如果想要实现加载更新，就只能访问后端，这时衍生出了web使用的脚本比如javascript，浏览器通过执行js脚本，配合dom可以在前端就实现页面更新效果，搭配ajax可以实现不阻塞当前页面的加载更新
    4. 前后端分离时期，原有的html页面都是在服务端渲染，前端获取的都是html页面数据，此后随着前端项目的日益复杂，前端走向独立的工程化道路，开始前后端分离，此时服务端不提供view层功能，前端不再直接获取html页面而是开始获取服务端的json数据，前端通过配合vue或react等js框架在前端完成html页面的渲染，至此前后端直接的交互变成了以json数据为主，类似于后端微服务的交互，也需要定义接口契约，前端开始走上独立发展的道路

html介绍

  1. 博纳斯李是万维网的创始人

    • 1991年编写了第一个静态网页的网站
    • 1994年创建了w3c联盟，用于制定网页开发规范

  2. 根据W3C规范，一个网页由三部分组成，分别使用html描述网页结构、css描述网页样式和js用于负责网页行为操作，这三种技术栈就是网页开发的三剑客

  3. 软件的开发方式分为两种，分别是CS和BS架构，S指的是后端服务器，C指的是Client，B指的是Browser，BS架构相较于CS架构的关键优势就在于跨平台，因为对于B来说，不管是哪个平台的B，其均是访问S的网页，所以具有天生的跨平台的特性，由于通过BS架构中的html、css和js技术栈，可以开发出丰富多彩的网页，所以前端开发主要采用BS架构，核心技术栈也就是html、css和js

  4. BS架构中，B使用的是http协议，但http协议本质上也是基于TCP协议，所以B从另一个角度来看其实也是C，但是对于开发者而言，在浏览器中其面向的就是http协议

html学习准备

  1. 使用webstorm或IDEA新建一个项目，在根目录下创建index.html，可以通过IDE快速打开网页，此时会IDE会启动一个默认绑定63342端口的web服务器，并打开浏览器访问该服务器，这种方式能模拟出服务环境，这不同于使用浏览器直接打开html文件采用的file协议
  2. 使用webstorm时，可以实现热部署从而快速获得页面效果，首先需要安装live edit插件(webstorm已经默认安装)，然后对IDE的debugger和live edit进行一些设置，比如刷新时间和端口等，接着在chrome浏览器中安装JetBrains IDE Support插件，并保证两者端口一致，这样在编写html网页的时候就可以通过debug该网页实现实时刷新，具体可参考博客(tips:windows中可以将窗口向左右一侧拖动，即可快速半屏)
  3. 详细查阅html标签可见html参考书

html基础

  1. html(hypertext markup language)超文本标记语言，其是一种描述网页内容结构的DSL，超文本的意思就是不仅仅是普通文本，还包含超链接和图片视频等，标记的意思是使用标签进行标记，用来表示不同的结构

  2. html元素表示从开始标签到结束标签的所有代码，html元素分为两种

    • 一种有元素内容，此时包括三部分，分别是开始标签、元素内容和结束标签(正斜杠位于标签字母前)
    • 另外一种是空元素，其并没有元素内容和结束标签，其只有开始标签，并且在开始标签的标签字母后有正斜杠/，比如换行<br/>

  3. html的注释，使用<!--这里就是注释-->，其不会被浏览器渲染显示

  4. html标签可以通过属性来提供更多标签功能，属性是通过键值对形式出现，键使用属性名表示，值必须是字符串，键与值之间用=分隔，键值对之间用空格分隔，属性分为普通属性和事件属性，

    1. 对于普通属性，比如比如所有标签都拥有的类名class(用于分类)、id(用于标识)、style(用于内联样式)等属性，以及标签特有的，比如a标签的target表示超链接跳转的方式

    2. 对于事件属性，可以设置事件响应代码，比如<body onclick="alert('hello html')">hello world</body>

  5. 这里介绍一个html网页的总体结构

    1. doctype网页声明，html网页首先需要声明html的版本，现在基本都使用2012年发布的html5，即<!DOCTYPE html>

    2. html网页是一个树形结构，最外围的标签就是html根标签，html标签自身需要声明一个属性lang即lang=en，然后里面包含着head头标签和body体标签

    3. head头标签用于进行网页的一些声明，一般不用于网页内容展示

      • 使用meta标签进行字符集声明，一般使用utf-8字符集，即<meta charset="UTF-8">，还可以通过name和content属性来为该网页添加一些元数据信息，方便搜索引擎检索
      • 使用title标签进行网页名称声明，比如<title>这是网页的名称</title>

    4. body体标签用于网页可见内容的展示，其标签内容会被浏览器渲染展示出来，这也是网页开发的主体部分

html常见标签

  1. 语义化标签，意思是代表网页的结构含义，注意这里需要关注的是语义，而不是其样式表现

    • h1h6标签用于标题展示，一般常用的是h1h3
  • p标签用于段落展示
    • q和blockquote引用标签，分别是行内引用和块引用

  1. html5的语义标签，其也代表特殊的结构含义，由于是html5新增所以使用频率较低

    • header标签表示头部、可以表示网页的头部也可以是各个区块的头部
  • footer标签表示底部
    • nav标签表示网页的导航
  • aside标签表示边栏
    • article标签表示一篇文章
  • main标签表示网页的主体部分，一般一个网页只有一个
    • section标签，表示一个区块，无语义

  1. br标签用于换行，hr标签用于水平线，它们是空元素标签

  2. img标签用于图片展示，一般在根目录下会创建images目录，用于存放图片文件

    1. src属性表示图片地址，比如<img src="images/w3c.jpg" >
    2. alt属性表示图片的描述文字，当图片无法显示时会显示，也用于搜索引擎收录
    3. img标签属于替换元素，其不属于行内元素和块元素的一种，其展示内容被资源所替代
    4. width和height属性来调整图片大小，如果只调整其中一个属性，那么图片就会按照比例扩缩，一般来说直接提供所需尺寸的图片，不进行扩缩，因为扩大图片可能会引起图片失真，缩小图片相比于使用小图片会占用更多内存
    5. 图片格式有多种，一般来说选择的依据是在效果和内存大小方面取得平衡
      • jpg(jpeg)，颜色较丰富，不支持动图和透明效果，一般用来显示照片
      • png，颜色丰富，支持透明效果
      • gif，支持动图，但颜色较少
      • webp，谷歌推出的图片格式，其兼具上述三种的所有优点，并且图片较小，缺点就在于对于老式浏览器兼容性较差

  3. 包括图片文件路径在内文件的路径分为两种，分别是绝对路径和相对路径

    • 相对路径，就是相对当前文件的路径

      .表示当前文件所在目录(当前文件就是书写该路径的文件)，..表示当前文件所在的上一级目录，文件名表示当前文件所在目录的文件，等价于./文件名，普通java项目的相对路径是从工程名开始计算，一般是classpath目录

    • 绝对路径，url地址，完整格式是http://ip:port/工程名/path，普通java项目中的绝对路径是盘符:/目录/文件名

  4. audio和video标签，用于音频和视频播放，与img标签类似，它也是替换元素，其也使用src属性引入文件，使用control属性表示允许用户控制播放，否则默认是不允许也就不展示

  5. a标签

    1. 超链接标签，用于当前页面跳转到另一个页面，跳转的页面可以是网络上的页面，也可以是相对当前页面的其他页面，href属性表示跳转的页面地址，比如<a href="http://www.baidu.com">百度网站</a>和<a href="jump.html">jump页面验证</a>
    2. 该标签可以嵌套除去a元素的其他任何元素，比如图片，比如<a href="index.html"><img src="images/w3c.jpg"></a>
    3. target属性表示文档被打开在何处，_self表示在当前tab中打开，默认值，_blank表示在新的tab打开
    4. a标签还可以实现锚点，通过href设置锚点值就可以跳转到该锚点，比如有一个锚点<a id="hello">这是一个hello锚点</a>，跳转的链接<a href="#hello">点击跳转到hello锚点</a>
    5. href属性还可以设置为javascript:;表示占位，此时点击该超链接没有效果

  6. iframe标签，框架标签用于为当前页面引入另一个页面，也就是嵌套页面，这里使用iframe标签，src属性表示嵌套的html页面的地址

  7. table标签

    • table标签，其有caption表格标题标签和tr行标签，其还拥有很多属性，比如border边框宽度、cellpadding单元格内边距、cellspacing单元格外边距

    • tr标签下有th标签和td标签，th标签表示列头单元格，td标签表示普通单元格

    • 在th和td标签上通过设置rowspan和colspan可以跨行跨列，对于多出的单元格需要删除

  8. 列表标签，用于聚合多个元素的集合标签，分为有序列表ol和无序列表ul

    1. 有序列表ol，列表选项li，默认使用数字表示序号，也可以通过ol标签的type属性设置为希腊数字i或者英文字母a，另外可以设置start属性，表示开始的数字

    2. 无序列ul表，列表选项也是li，默认形状是空心圆disc，可以通过type属性设置为circle实心圆或者square实方形

    3. 列表可以进行嵌套，从而实现多层级，这里嵌套的方式就是直接在需要展开的li标签后面写上新的ul或ol列表即可

  9. form标签，表单标签，以上的标签只能用于展示，而表单标签可以用于输入和提交数据，其拥有多种类型的输入子标签

    1. input标签，其用于常规输入，其为空元素标签，通过type可以指定输入类型，

      • text文本标签，通过value属性指定默认文本
      • password密码，通过value属性指定默认密码，其为掩码不展示
      • radio单选框，需要使用name属性表示同一组产生互斥，通过checked=”checked”表示默认选项
      • checkbox多选框，通过checked=”checked”表示默认勾选选项
      • submit提交按钮，通过value属性进行指定按钮名称
      • reset重置按钮、通过value属性进行指定按钮名称
      • button普通按钮，通过value属性进行指定按钮名称，按钮也可以通过单独的button标签来表示，其type是submit或reset，其按钮名称则是通过闭合的元素内容表示，不推荐使用
      • hidden隐藏，通过value属性指定隐藏信息，其并不会展示仅用于发送信息
      • file文件按钮，表示提交的文件

    2. select标签，表示下拉栏，下拉选项是option子标签，使用selected=”selected”表示默认选择项

    3. textarea标签，表示文本区域，用于多行输入，可以使用rows和cols设置宽高，通过显式设置标签内容表示默认显示的内容

    4. form标签拥有若干个属性，

      • action属性表示提交表单数据，发送http请求的url地址，默认是当前页面的url
      • method属性表示提交数据发送的http方法，默认是get方法，使用get提交表单数据，其会将表单数据会通过url参数的形式提供，所以其不够安全而且受到长度限制，使用post提交，则通过请求体body来传递表单数据，更加安全并且长度也不受限制

    5. 表单提交时，需要将表单信息发送到服务器，所以对于每一项数据是需要以键值对形式传递

      • 使用name属性标识每个上传数据的名称，其中对于单选框、多选框和下拉栏仅需要一个name属性设置，
      • 使用value属性表示上传数据的值，其中对于单选框、多选框和下拉栏需要显式设置value属性，而其他输入框由于存在手动输入，会将其输入作为默认的value属性值上传而不需要设置

    1. link其是空元素标签，仅能在head标签下使用，可以出现多次，常用来引入css样式表

    2. rel属性表示当前文件与引入文件的关系，type属性表示引入文件的类型，href属性表示引入文件的URL，比如

  11. script标签，脚本标签用于引入脚本，有两种方式来为页面引入js脚本，第一种是在script标签内书写js代码，另一种使用script标签的src属性引入js文件，这两种方式类似于内部样式表和外部样式表

  12. 实体，html中一些符号是作为关键字存在，如果想要显示它们，需要进行转义，

    • 对于<使用&lt;，对于>使用&gt;
    • 对于空格，使用&nbsp;，注意在html里面如果有连续多个空格，会被浏览器自动截短至一个空格

  13. 块元素

    1. html的标签中标题h1~h6、段落p、列表ul和div都是块级元素(block element)，其均会保证独占一行，并且对于标题标签、段落标签和列表标签，其会在上方或下方各空出一行，但如果已经存在块级标签的空行则不再空行，div一般作为块级容器存在
    2. 与块级元素相反的就是内联元素(inline element)，其并不会独占一行，比如a、input和span标签，其中span作为一种内联容器存在
    3. 块级元素用于对于页面的布局，行内元素用于对文字的包裹，一般来说，块级元素内可以放块级元素和行内元素(p元素不能存放块级元素)，行内元素不会放块级元素

  14. 浏览器会对部分不符合规范的html代码进行修正，比如在html标签外部写标签、在head标签内写展示内容、在p元素内使用块元素等，对于这些问题，浏览器会将其修正，使得仍然能够正常展示，这里需要注意的是浏览器不会修改网页源代码，其修改的是网页的内存布局dom树

xhtml和html5介绍

  1. xhtml是通过xml来编写html，由于html松散的语法规范，使得存在很多badcode，而由于xml的严格的规范特性，使得使用xhtml编写网页代码会更加复合规范，目前xhtml得到浏览器的广泛支持，xhtml中的规范列举如下，

    • 首先得进行xhtml文档声明，XHTML <!DOCTYPE>，此时该网页就是xhtml网页
    • 标签元素必须小写且必须嵌套，并始终关闭
    • 标签的属性必须小写，属性值必须使用引号包围，另外属性不得简写，比如<input checked />是错误的，<input checked="checked" />是正确的

  2. html5是目前最新的html标准，其标签更加丰富

    1. 添加了一些表单类型比如时间、视频video和语义标签等，其中语义标签的意思是，对于div和span标签都是非语义的，不表示任何含义，通常在网页布局时，会通过id属性来标识div各个作用区域，在html5通过语义标签，能从标签名称就能知道标签内容的作用与含义，比如article、header、nav、footer、aside、section等标签

    2. 添加一些新通用属性，比如contentEditable，表示是否可编辑，hidden表示隐藏，其页面不可见，spellcheck属性表示会对输入内容进行检查

  3. html最佳实践，任何最佳实践都是为了规范统一，方便阅读，对于代码的要求不仅仅是可运行，更在于可阅读和可维护，最佳实践请见参考文档
